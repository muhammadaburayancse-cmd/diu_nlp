The Exam Question
Read this carefully. We will solve every part with math.
Question

You are given 3 short documents and asked to run LDA with K = 2 topics using Gibbs Sampling.


Document 1 (D1):  "fish lion star"

Document 2 (D2):  "lion jungle star"

Document 3 (D3):  "fish coral star"


Use hyperparameters α = 1 and β = 1.


(a) Show the random initialization of topic assignments.

(b) Write the Gibbs Sampling update formula.

(c) Compute the probabilities for word "fish" in D1 after one iteration.

(d) Show what changes and why after reassignment.

(e) After convergence, what do the topic-word and document-topic distributions look like?

What we have

Vocabulary (V = 5 words)

fish lion star jungle coral

Settings

K = 2 topics α = 1 β = 1 3 documents 9 words total
Why α = 1 and β = 1?
These are Dirichlet smoothing values. α = 1 means "we expect documents to mix topics evenly at the start — no strong prior." β = 1 means "we expect topics to use all words equally at the start." They prevent division-by-zero and allow unseen combinations. Larger values = smoother, more uniform distributions.

Step 3
The Core Formula
Answer (b) — The Gibbs Sampling update equation. Understand every symbol.
The formula
P(z_i = k | all other assignments) ∝ (n_dk + α) × (n_kw + β) / (n_k + V·β)
What every symbol means
SymbolMeaningIn plain English
z_i = kWord i is assigned to topic kWhich topic does this word belong to?
n_dkWords in doc d assigned to topic kHow popular is topic k in this document?
αDocument smoothing (= 1)Prevents zero probability; adds a "floor"
n_kwTimes word w assigned to topic kHow often does topic k use this word?
βWord smoothing (= 1)Prevents zero probability for rare words
n_kTotal words in topic kHow big is topic k overall?
V·βVocabulary size × β = 5 × 1 = 5Normalisation constant
∝"Proportional to" (not equals)We compute raw scores, then normalise
MINUS current wordWe REMOVE word i from counts firstDon't count the word we're updating!
The two-part intuition
Part 1 — Document preference
(n_dk + α)
"How much does this document already use topic k?"
If D1 already has 5 words assigned to Topic 0, it probably wants topic 0 for the next word too.
Part 2 — Word-topic affinity
(n_kw + β) / (n_k + Vβ)
"How much does topic k like this word?"
If "fish" appeared 10 times in Topic 0 words, Topic 0 probably owns "fish".
Key step before computing
Before we calculate the probability for word i, we REMOVE word i from the counts. This is the "all other assignments" part. Then we recalculate with word i excluded, pick the topic, re-add the word to the winning topic's counts.
Step 4
Iteration 1 — Updating "fish" in D1
Answer (c) — Full probability calculation, every number shown.
Starting state (from random init)
Current assignment of "fish" in D1
"fish" in D1 is currently assigned to Topic 0. We REMOVE it from the counts before recalculating.

After removing "fish" in D1 from Topic 0, the updated counts are:

Topic 0Topic 1
D1 (n_dk)1  (was 2, minus 1)1
fish (n_kw)0  (was 1, minus 1)1
Total n_k4  (was 5, minus 1)4
Now plug into the formula
P(z = k | all others) ∝ (n_dk + α) × (n_kw + β) / (n_k + V·β) V = 5, α = 1, β = 1, so V·β = 5×1 = 5

For Topic 0:

n_d1,0 (after remove) = 1  |  n_0,fish (after remove) = 0  |  n_0 (after remove) = 4
(1 + 1) × (0 + 1) / (4 + 5) = 2 × 1 / 9 = 2/9 ≈ 0.222

For Topic 1:

n_d1,1 (unchanged) = 1  |  n_1,fish (unchanged) = 1  |  n_1 (unchanged) = 4
(1 + 1) × (1 + 1) / (4 + 5) = 2 × 2 / 9 = 4/9 ≈ 0.444
Normalise so they sum to 1
Raw scores: Topic 0 = 2/9, Topic 1 = 4/9. Sum = 6/9
P(Topic 0) = (2/9) / (6/9) = 2/6 = 0.333 (33.3%)
P(Topic 1) = (4/9) / (6/9) = 4/6 = 0.667 (66.7%)
Decision
Topic 1 has probability 66.7%. We sample from this distribution (or just pick the higher one). "fish" in D1 gets reassigned to Topic 1.
Update all counts after reassignment
Topic 0Topic 1
D1 (n_dk)12  (+1 because fish moved here)
fish (n_kw)02  (+1)
Total n_k45  (+1)

These updated counts are used for the NEXT word's calculation.

Step 5
Iteration 1 — Updating "star" in D1
Same process. Notice how changed counts affect the result.
Current state (after "fish" was moved to Topic 1)

"star" in D1 is currently Topic 0. Remove it first:

Topic 0Topic 1
D1 (n_dk)0  (was 1, remove star)2
star (n_kw)1  (was 2, remove this star)1
Total n_k3  (was 4, minus 1)5
Calculate probabilities

For Topic 0:

(0 + 1) × (1 + 1) / (3 + 5) = 1 × 2 / 8 = 2/8 = 0.25

For Topic 1:

(2 + 1) × (1 + 1) / (5 + 5) = 3 × 2 / 10 = 6/10 = 0.6
Normalise
Sum = 0.25 + 0.6 = 0.85
P(Topic 0) = 0.25 / 0.85 = 0.294 (29.4%)
P(Topic 1) = 0.60 / 0.85 = 0.706 (70.6%)
Decision
"star" also moves to Topic 1 (70.6% probability). Why? D1 now leans strongly toward Topic 1 (n_d1,1 = 2), so words in D1 are pulled that way.
Why did counts change from Step 4?
Because "fish" was already moved to Topic 1. n_d1,1 went from 1 to 2. This INCREASED the pull of Topic 1 for "star". This is the cascade effect — each reassignment influences the next word's probability. This is why Gibbs Sampling works: consistent words reinforce each other's topic assignments.

Complete Answer Summary

(a) Init:Assign each word a random topic. Build n_dk and n_kw tables from these.
(b) Formula:P(z=k) ∝ (n_dk + α) × (n_kw + β) / (n_k + Vβ)
(c) P("fish"):Topic 0 = 0.333, Topic 1 = 0.667 → assign Topic 1
(d) Changed:fish: T0→T1, star: T0→T1, lion: T1→T0. Counts cascade.
(e) Final φ:T0 = {lion:0.33, jungle:0.22}, T1 = {star:0.40, coral:0.20}
(e) Final θ:D3 is 80% Topic 1; D2 is 60% Topic 0; D1 is 60% Topic 1
The one-line exam definition
LDA Gibbs Sampling repeatedly removes one word's topic assignment and resamples it using P ∝ (doc-topic affinity) × (word-topic affinity), updating the count tables after each reassignment, until assignments converge.
