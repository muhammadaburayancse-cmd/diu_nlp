Introduction

Welcome to NLP Fundamentals

Natural Language Processing (NLP) is the subfield of Artificial Intelligence concerned with enabling computers to understand, interpret, and generate human language. From the search engine that completes your query to the assistant that reads your email aloud, NLP is invisible infrastructure.

This lecture note covers four tightly interconnected topics that form the preprocessing backbone of almost every NLP system. Master these, and every advanced topic — transformers, LLMs, sentiment analysis — becomes far easier to reason about.

📖 Professor's Opening Note

Every NLP system, regardless of how sophisticated, still begins with raw text. Understanding how that raw text is cleaned, structured, and reduced into meaningful units is not a "nice-to-have" — it is fundamental. Students who gloss over preprocessing consistently produce NLP systems that fail in production. We will be thorough here.

Topic 1

The NLP Pipeline Taxonomy

An NLP pipeline is a structured, sequential set of processing stages that transform unstructured natural language text into a structured representation that machines can reason about. Rather than treating a document as a flat string, the pipeline decomposes it into progressively richer layers of linguistic knowledge.

📌 Formal Definition

NLP Pipeline: A layered transformation process in which raw text passes through a series of analytical modules — each module consuming the output of the previous one — to build structured linguistic representations at morphological, syntactic, and semantic levels.

The Standard Pipeline Architecture

Tokenization
→
Normalization
→
Morphological Analysis
→

Output / Representation
📖 Professor's Note — Pipeline Taxonomy

Notice that each stage depends on the stage before it. If tokenization is incorrect, every downstream module will amplify that error. This is called error propagation and it is the central challenge in cascaded NLP pipelines. Modern neural approaches (BERT, GPT) attempt to solve this by learning jointly across levels, but classical pipelines remain essential for interpretability and low-resource settings.

1a. Morphological Analysis

Morphology is the branch of linguistics concerned with the internal structure of words. In NLP, morphological analysis is the process of decomposing a word into its constituent morphemes — the smallest units of meaning — and determining the word's grammatical properties.

📌 Definition — Morpheme

A morpheme is the smallest meaningful unit of a language. For example: unhappiness = un- (prefix, negation) + happy (root) + -ness (suffix, nominaliser).

Types of Morphological Processes

ProcessDescriptionExample
InflectionGrammatical variation that does not change the core meaning or lexical categoryrun → running, runs, ran
DerivationCreates a new word, often changing the lexical category or meaninghappy → unhappy, happiness
CompoundingTwo or more free morphemes combine to form a new wordblack + board → blackboard
CliticisationA morpheme attaches phonologically to an adjacent wordI am → I'm; he will → he'll

Morphological Analysis Tasks in NLP

  • Morphological segmentation: splitting a word into morphemes (e.g., played → play + ed)
  • POS tagging: assigning grammatical categories (noun, verb, adjective…)
  • Lemmatization / stemming: reducing word forms to a base (detailed in Topic 3)
  • Morphological generation: producing inflected forms from a lemma (used in NLG systems)
🧪 Real-World Example

Search engines use morphological analysis so that a query for running also retrieves documents containing run, runs, runner. Without it, results would be severely limited by surface-form mismatches.

# Morphological analysis with NLTK
import nltk
from nltk.stem import WordNetLemmatizer

lemmatizer = WordNetLemmatizer()

words = ["running", "unhappiness", "studies", "better"]
for w in words:
    print(f"{w:15} → lemma: {lemmatizer.lemmatize(w, pos='v')}")

# Output:
# running         → lemma: run
# unhappiness     → lemma: unhappiness
# studies         → lemma: study (pos='n')
# better          → lemma: well  (pos='a')

1b. Syntactic Analysis (Parsing)

Syntactic analysis, often called parsing, is the process of determining the grammatical structure of a sentence by analyzing the relationships among its words. The output is typically a parse tree or a dependency graph.

📌 Definition — Parse Tree

A parse tree (also called a constituency tree or phrase-structure tree) is a hierarchical, tree-shaped representation of the syntactic structure of a sentence according to a formal grammar. Each internal node represents a grammatical category (NP, VP, PP…) and each leaf node represents a terminal token (word).

Two Major Approaches to Parsing

📐 Constituency Parsing (Phrase-Structure)

Groups words into nested constituents (phrases). Based on Context-Free Grammars (CFG). Each node is labeled with a phrase type: NP, VP, PP, etc.

Output: A hierarchical tree structure (Penn Treebank format).

🔗 Dependency Parsing

Represents grammatical structure as a set of binary, typed dependencies between words. One word is the head; the other is the dependent.

Output: A directed graph, one node per token (Universal Dependencies format).

🧪 Technical Example — Constituency Parse

Sentence: "The quick fox jumped over the fence."

(S
  (NP (DT The) (JJ quick) (NN fox))
  (VP (VBD jumped)
    (PP (IN over)
      (NP (DT the) (NN fence)))))

NP = Noun Phrase, VP = Verb Phrase, PP = Prepositional Phrase, DT = Determiner, JJ = Adjective, NN = Noun, VBD = Verb past tense, IN = Preposition

🧪 Technical Example — Dependency Parse

Sentence: "The cat sat on the mat."

sat     ROOT
cat     nsubj  → sat
The     det    → cat
on      prep   → sat
mat     pobj   → on
the     det    → mat
🎯 Exam Tip

Be able to draw a constituency parse tree for a short sentence. Examiners often provide a sentence and ask you to label the tree nodes with grammatical category symbols (S, NP, VP, PP, DT, NN, VB…). Also know the difference between ambiguity types: structural ambiguity (same words, different trees) vs. lexical ambiguity (same word, different meanings).

1c. Semantic Analysis

Semantic analysis is the process of determining the meaning of a sentence or discourse, moving beyond structural form. While syntax asks "is this grammatically well-formed?", semantics asks "what does it actually mean?".

📌 Definition — Compositional Semantics

Compositional semantics (Frege's Principle) holds that the meaning of a complex expression is determined by the meanings of its parts and the syntactic rules used to combine them. It provides the theoretical foundation for semantic parsing in NLP.

Levels of Semantic Analysis

LevelWhat It CapturesKey NLP TaskExample
Lexical SemanticsMeaning of individual wordsWord Sense Disambiguation (WSD)"bank" (river vs. financial)
Sentential SemanticsMeaning of a full sentenceSemantic Role Labeling (SRL)Who did what to whom
Discourse SemanticsMeaning across sentencesCo-reference Resolution"She said she was tired." → who is she?
PragmaticsMeaning in context / intentIntent Detection"Can you pass the salt?" = a request, not a question
🧪 Real-World Example — Semantic Role Labeling

Sentence: "NASA launched the satellite into orbit."

  • Agent (ARG0): NASA — the one performing the action
  • Predicate: launched — the event/action
  • Theme (ARG1): the satellite — what was acted upon
  • Location (ARGM-LOC): into orbit — where it went
⚠️ Common Misconception

Students often confuse syntax and semantics. Remember: a sentence can be syntactically correct but semantically anomalous — Chomsky's famous example: "Colorless green ideas sleep furiously." Grammatically perfect; semantically nonsensical. The pipeline levels are distinct.

Topic 2

Text Normalization

Text normalization is the collection of preprocessing operations that transform raw, heterogeneous text into a cleaner, more uniform form suitable for downstream linguistic analysis. Think of it as standardizing the "surface form" of language before deeper analysis begins.

📌 Definition

Text normalization refers to the systematic application of transformations to a text corpus to reduce superficial variation that does not contribute to meaning — such as inconsistent casing, punctuation noise, encoding artifacts, or typographic errors — while preserving semantically relevant distinctions.

Core Normalization Operations

OperationWhat It DoesExample
TokenizationSplits text into tokens (words, subwords, chars)"Hello!" → ["Hello", "!"]
Case-foldingConverts all characters to lowercase"NASA" → "nasa"
Punctuation removalStrips non-alphanumeric characters"it's" → "its" or "it s"
Number normalizationReplaces digits with a token or normalizes forms"$1,200" → "NUM" or "1200"
Accent strippingRemoves diacritics from characters"naïve" → "naive"
Whitespace normalizationCollapses multiple spaces/tabs/newlines"hello  world" → "hello world"

2a. Deterministic String Tokenization Strategies

Tokenization is the first and most fundamental normalization step. Before any linguistic analysis can occur, we must define what the basic unit of processing is. A token is that basic unit — usually a word, punctuation mark, or number.

📌 Definition

Tokenization is the process of segmenting a raw text string into a sequence of tokens. A deterministic tokenizer uses explicit, pre-defined rules (typically regular expressions or finite-state automata) rather than probabilistic models, ensuring the same input always yields the same output.

Strategy 1 — Whitespace Tokenization

The simplest strategy: split on every whitespace character. Fast but naive.

# Python: whitespace tokenization
text = "She said, 'Don't panic!' and left."
tokens = text.split()
print(tokens)
# ["She", "said,", "'Don't", "panic!'", "and", "left."]
# Problem: punctuation is stuck to words!

Strategy 2 — Regular Expression Tokenization

Define a regular expression that matches tokens (positive match) or delimiters (split pattern). This is the dominant deterministic approach in classical NLP.

import re

# Positive token pattern: match words, numbers, and punctuation separately
pattern = r"\w+|[^\w\s]"
text = "I can't stop thinking about U.S. foreign policy (2024)!"
tokens = re.findall(pattern, text)
print(tokens)
# ['I', 'can', "'", 't', 'stop', 'thinking', 'about',
#  'U', '.', 'S', '.', 'foreign', 'policy', '(', '2024', ')', '!']

# Better: preserve contractions
pattern2 = r"[A-Za-z]+(?:'[a-z]+)?|\d+(?:\.\d+)?|[^\w\s]"
tokens2 = re.findall(pattern2, text)
print(tokens2)
# ["I", "can't", "stop", "thinking", "about", "U", "S", "foreign", "policy", "2024", "!"]

Strategy 3 — Rule-Based Tokenizer (NLTK TreebankWordTokenizer)

Encodes a hand-crafted set of rules for English, handling apostrophes, hyphens, abbreviations, and punctuation consistently.

from nltk.tokenize import TreebankWordTokenizer, word_tokenize

tokenizer = TreebankWordTokenizer()
text = "Don't you know Mr. Smith's theory? It costs $5.99!"
print(tokenizer.tokenize(text))
# ['Do', "n't", 'you', 'know', 'Mr.', 'Smith', "'s", 'theory', '?',
#  'It', 'costs', '$', '5.99', '!']

Strategy 4 — Character n-gram / Byte-Pair Encoding (BPE)

Modern subword tokenization: not purely deterministic in the classic sense, but the merge rules learned during training are applied deterministically at inference time.

# BPE example (conceptual, as used in GPT-style models)
# "unhappiness" might be tokenized as:
# ["un", "happ", "iness"]  ← subword units
# This solves the out-of-vocabulary (OOV) problem by
# decomposing rare words into known subword pieces.
🎯 Exam Tip — Tokenization Edge Cases

Examiners love edge cases. Be prepared to discuss:

  • Contractions: "don't" → ["do", "n't"] vs. ["don", "'", "t"] — different tokenizers disagree
  • Hyphenated words: "state-of-the-art" — one token or four?
  • Abbreviations: "U.S.A." — the trailing period is end-of-sentence or abbreviation marker?
  • URLs & email addresses: should user@mail.com be kept as one token?
  • Multiword expressions: "New York" — two tokens or one named entity?
  • Non-English scripts: Chinese and Japanese have no whitespace — require dictionary-based segmentation (e.g., Jieba for Chinese).

2b. Case-Folding and Its Limits

Case-folding is the normalization of all characters in a text to a uniform case — almost always lowercase — to eliminate spurious distinctions caused by capitalisation conventions rather than semantic intent.

📌 Definition

Case-folding is a text normalization operation that maps every character in a string to its lowercase equivalent using a deterministic character-level mapping, thereby collapsing distinctions that arise solely from typographic or positional capitalisation conventions (sentence-initial capitalisation, ALL-CAPS emphasis, etc.). 

# Python case-folding
text = "The PRESIDENT of the United States visited NASA HQ."
folded = text.lower()
print(folded)
# "the president of the united states visited nasa hq."

# Python's casefold() is more aggressive than lower() — handles Unicode better
print("Straße".casefold())   # → "strasse"  (German sharp-s → ss)
print("Straße".lower())     # → "straße"   (lower() keeps ß)

Why Case-Folding Is Beneficial

  • Vocabulary reduction: "The" and "the" are no longer two types — reduces vocabulary size (V) and data sparsity
  • Consistent matching: A search for "london" retrieves "London" and "LONDON"
  • Generalization: Models don't need extra parameters to represent capitalised duplicates
🧪 Example — Vocabulary Reduction Effect

In a typical English corpus, approximately 5–10% of vocabulary types are "case duplicates" — identical tokens differing only in capitalisation. For a vocabulary of 100,000 types, case-folding can reduce this to ~90,000–95,000, meaningfully reducing model size and improving the count estimates used in n-gram language models.

⚠️ The Limits of Case-Folding

Case-folding is not without significant dangers. In several important situations, capitalisation is semantically contrastive, meaning that folding to lowercase destroys information.

ScenarioOriginal FormAfter Case-FoldingInformation Lost
Proper Nouns vs. Common Nouns "Turkey" (country) vs. "turkey" (bird) both → "turkey" Named entity boundary
Named Entity Recognition "Apple" (company) vs. "apple" (fruit) both → "apple" Organisation vs. common noun
Acronyms "IT" (Information Technology) vs. "it" (pronoun) both → "it" Acronym identity
Scientific terminology "AIDS" (disease) vs. "aids" (verb, helps) both → "aids" Medical term vs. verb
Sentiment emphasis "I LOVE this!" vs. "I love this!" identical after folding Intensity signal
Social media discourse "#MeToo" vs. "#metoo" both → "#metoo" Hashtag convention
⚠️ Critical Limitation for NER Systems

In Named Entity Recognition (NER) and Information Extraction tasks, case information is a primary feature. Capitalisation in English strongly correlates with proper-noun status. Applying case-folding before NER will substantially degrade model performance. The general principle: apply case-folding only when the downstream task does not require distinction between proper and common nouns.

📖 Professor's Note — When to (Not) Case-Fold

The decision to case-fold must be task-driven. For information retrieval and bag-of-words classification tasks, case-folding is almost always beneficial. For NER, co-reference resolution, machine translation, and sentiment analysis, preserve case information as a feature. In modern transformer-based models, the tokenizer is trained on cased text and handles this automatically — but understanding the underlying trade-off is essential for system design.

Topic 3

Stemming vs. Lemmatization

Both stemming and lemmatization are morphological normalization techniques whose goal is the same: reduce different surface forms of a word to a single canonical form, thereby collapsing inflectional variation and improving matching. However, they differ fundamentally in their approach, linguistic sophistication, and the quality of the canonical form they produce.

📌 Definitions

Stemming: A heuristic process that chops or modifies the suffix of a word using a set of simple string-transformation rules, producing a stem — a truncated, often non-standard form that is not necessarily a valid dictionary word.

Lemmatization: A linguistically informed process that reduces a word form to its canonical dictionary form — its lemma — through morphological analysis and vocabulary lookup, always producing a valid word.

3a. The Porter Stemmer — Rules and Mechanics

The Porter Stemmer (Martin Porter, 1980) is the most widely studied and used English stemmer. It applies a cascade of string-transformation rules in five sequential phases (Steps 1a, 1b, 1c, 2, 3, 4, 5a, 5b). Each step is applied in order; the first matching rule in each step fires and the remaining rules are skipped.

📌 The Measure (m) — Core Concept of the Porter Stemmer

The Porter Stemmer introduces a formal notion of word measure, denoted m, which counts the number of vowel-consonant sequences in the stem preceding the suffix being removed.

Let C = one or more consonants, V = one or more vowels. Then:

[C] (VC)^m [V] defines a word with measure m.

Examples: "tree" → m=0; "trouble" → m=1; "oaten" → m=1; "troubles" → m=1; "private" → m=2

⚠️ Known Pathologies of the Porter Stemmer

Overstemming: different words collapse to the same stem (conflation errors)

  • "generalizing" and "general" → both stem to "general" (correct)
  • "university" and "universe" → both stem to "univers" (incorrect — different meanings!)

Understemming: related forms are not collapsed to the same stem

  • "alumnus", "alumni" → stemmed to "alumnus" and "alumni" (not normalized!)

3b. WordNet Morphological Analysis & Lemmatization

WordNet (George Miller et al., Princeton, 1985–) is a large-scale lexical database for English. It groups words into sets of synonyms called synsets, each representing a distinct lexical concept, and records semantic relations among synsets (hypernymy, hyponymy, meronymy, antonymy).

📌 Definition — Lemma

A lemma is the canonical, citation, or base form of a word as it would appear in a dictionary: the infinitive form of verbs ("run" not "running"), the singular form of nouns ("mouse" not "mice"), and the positive degree of adjectives ("good" not "better" or "best").

How WordNet Lemmatization Works

  1. POS tagging: The input word form is tagged with its part of speech (NN, VBG, JJ, etc.).

  2. Morphological exception lookup: The (word, POS) pair is looked up in WordNet's exception lists — hand-curated tables of irregular forms (e.g., "mice → mouse", "went → go", "better → good").

  3. Algorithmic detachment: If no exception match is found, a set of ordered detachment rules is applied to the suffix based on the POS tag to produce candidate stems.

  4. Vocabulary validation: Each candidate stem is checked against the WordNet lexicon. Only valid dictionary words are returned.

  5. Lemma returned: The first valid base form found is returned as the lemma.

from nltk.stem import WordNetLemmatizer
from nltk.corpus import wordnet

wnl = WordNetLemmatizer()

test_cases = [
    ("runs",    "v"),
    ("running", "v"),
    ("ran",     "v"),
    ("mice",    "n"),
    ("geese",   "n"),
    ("better",  "a"),   # adjective
    ("best",    "a"),
    ("corpora", "n"),
    ("happily", "r"),   # adverb
]

print(f"{'Word':12} {'POS':4} → {'Lemma':12}")
print("-" * 32)
for word, pos in test_cases:
    lemma = wnl.lemmatize(word, pos=pos)
    print(f"{word:12} {pos:4} → {lemma:12}")

# Word         POS  → Lemma
# runs         v    → run
# running      v    → run
# ran          v    → run
# mice         n    → mouse
# geese        n    → goose
# better       a    → good
# best         a    → good
# corpora      n    → corpus
# happily      r    → happily   ← Note: adverb lemmatization is limited

📖 Professor's Note — Choosing Between Them

The choice is fundamentally a precision-speed trade-off. If you are building a large-scale search engine that indexes millions of documents in real time, the raw throughput advantage of stemming is often decisive. If you are building a medical record extraction system, a chatbot, or any application where a non-word output would propagate errors downstream, lemmatization is the correct choice. In modern deep-learning NLP pipelines (BERT, GPT), neither is applied — subword tokenization (BPE, WordPiece) replaces both — but understanding them is essential for classical NLP and for interpreting legacy systems.

🎯 Exam Tip

A classic exam question: "Apply the Porter Stemmer to the word 'generalizing' and show each step." Practice tracing through the steps manually. Also be ready to contrast specific examples where stemming fails but lemmatization succeeds, especially with irregular forms.

Topic 4

Stop-Word Filtration

Stop-word filtration is a text preprocessing technique that removes a predefined set of highly frequent, semantically weak function words from a text before further processing. The intuition is straightforward: words like "the", "is", "at", "which", and "on" appear in almost every document and carry little discriminative power for most tasks.

📌 Definition

Stop words are a set of commonly occurring words in a language that are typically removed before a text is processed in natural language processing tasks, because they carry minimal semantic information and high frequency renders them uninformative for tasks such as document classification, information retrieval, and topic modelling.

🧪 Example — What Stop Words Look Like

NLTK English stop-word list (subset): a, an, the, is, are, was, were, be, been, being, have, has, had, do, does, did, will, would, could, should, may, might, shall, must, can, that, this, these, those, i, me, my, myself, we, our, you, your, he, she, it, they, them, his, her, its, their, at, by, for, in, of, on, to, with, about, after, before, between, during, from, into, through, under, until, up, about, against, among...

The Mechanics of Stop-Word Filtration

  1. Tokenize the input text into a sequence of tokens.

  2. Load the stop-word list — either a standard list (NLTK, spaCy, Gensim) or a custom domain-specific list stored as a set for O(1) lookup.

  3. Filter: For each token, check membership in the stop-word set. Retain the token if it is NOT in the stop-word set.

  4. Reconstruct the filtered token sequence for downstream processing.

from nltk.corpus import stopwords
from nltk.tokenize import word_tokenize

# Load stop words as a set (O(1) lookup)
stop_words = set(stopwords.words('english'))

text = "The researchers at the university discovered that machine learning models perform well on large datasets."
tokens = word_tokenize(text.lower())

# Filter: keep only non-stop-words
filtered = [t for t in tokens if t not in stop_words and t.isalpha()]

print("Original:",  tokens)
print("Filtered:",  filtered)
print(f"Reduction: {len(tokens)} → {len(filtered)} tokens ({(len(tokens)-len(filtered))/len(tokens)*100:.0f}% removed)")

# Original: ['the', 'researchers', 'at', 'the', 'university', 'discovered', 'that',
#             'machine', 'learning', 'models', 'perform', 'well', 'on', 'large', 'datasets', '.']
# Filtered: ['researchers', 'university', 'discovered', 'machine', 'learning',
#             'models', 'perform', 'well', 'large', 'datasets']
# Reduction: 16 → 10 tokens (37% removed)

Rules and Regularization of Stop-Word Lists

There is no universal stop-word list — the "correct" list depends on the domain, language, and task. Several principles govern their construction:

PrincipleExplanationExample
Frequency-based Words above a corpus-specific frequency threshold are treated as stop words (e.g., top 1% by TF) "the" appears in ~90% of English documents
Low IDF Words with very low Inverse Document Frequency are removed (they appear in too many documents to be discriminative) IDF("the") ≈ 0 across any large corpus
POS-based Remove all determiners (DT), prepositions (IN), conjunctions (CC), and pronouns (PRP) by default "a", "an", "the", "of", "in", "and", "but"
Domain-specific extension Add domain-frequent terms that are non-discriminative within the domain (e.g., "patient", "hospital" in a medical corpus) In medical NLP: "patient" may be added to stop list
Hand-curated lists Expert linguists manually review and curate, balancing recall and specificity NLTK: 179 English stops; spaCy: ~326 by default

Advantages of Stop-Word Filtration

  • Dimensionality reduction: Reduces feature space significantly (often 30–50% of tokens are stop words), making models faster and more memory-efficient.
  • Noise reduction: Removes tokens that add noise without adding signal for classification, clustering, or retrieval tasks.
  • Improved TF-IDF performance: Stops words would otherwise dominate term-frequency scores; removing them allows content words to dominate.
  • Computational efficiency: Smaller vocabulary → smaller matrices in Bag-of-Words and TF-IDF models, faster training.
  • Better topic modeling: LDA and other topic models produce more interpretable topics when stop words are removed.

Limitations and Dangers of Stop-Word Filtration

LimitationExplanationTask Affected
Semantic negation loss "not good" → "good" after removing "not" — completely reverses meaning Sentiment Analysis
Question type loss Removing "who", "what", "where", "when", "why", "how" destroys intent signals QA, Intent Detection
Named entity fragmentation "The Who" (band name) → "Who" or nothing; "to be or not to be" → empty NER, Literary Analysis
Discourse coherence Connectives ("however", "therefore", "although") signal logical relations between clauses Discourse Analysis, Summarization
Language model perplexity LMs require natural language including function words; remove them and fluency collapses Language Generation
Cross-lingual issues Stop-word lists are language-specific; applying an English list to another language is harmful Multilingual NLP
🧪 Classic Failure Case — Sentiment Reversal
text_pos = "The movie is not bad at all."
text_neg = "The movie is bad."

# After stop word removal + stemming:
filtered_pos = ["movi", "bad"]    # "not" removed! Appears negative!
filtered_neg = ["movi", "bad"]    # Identical representation!

# A bag-of-words model sees these as IDENTICAL sentences.
# This is a fundamental problem for negation-sensitive tasks.

This illustrates why modern sentiment analysis systems either retain negation markers or use n-gram features that preserve local context ("not_bad" as a bigram feature).

📖 Professor's Note — The Task-Dependency Principle

A central lesson of NLP preprocessing: there is no universally correct preprocessing strategy. Every preprocessing decision — whether to case-fold, whether to stem or lemmatize, whether to remove stop words — must be evaluated against the target task. In modern deep-learning NLP, many of these decisions are made implicitly by the tokenizer and the model architecture. But in classical ML-based NLP (SVM, Naive Bayes, logistic regression on BoW features), these choices dominate performance. Understanding them is essential for both exam success and real-world system design.

🎯 Exam Tip — Stop Words and Task Context

Examiners frequently ask: "Should stop words be removed for task X? Justify your answer." Always answer with reference to:

  • Whether function words carry task-relevant information (negation, discourse, question type)
  • Whether the model architecture already handles high-frequency words (e.g., TF-IDF downweights them automatically)
  • Whether the vocabulary size constraint justifies the information loss

For IR / text classification / topic modelling: usually remove. For sentiment, QA, NMT, dialogue: usually retain.

Summary

Key Takeaways

NLP Pipeline

Processing moves through morphological → syntactic → semantic layers. Error propagation is the core challenge of cascaded pipelines.

Morphological Analysis

Decomposes words into morphemes. Enables stemming, lemmatization, and POS tagging. Critical for morphologically rich languages.

Syntactic Analysis

Two paradigms: constituency (phrase structure trees) and dependency (typed binary relations). Syntactic ambiguity is a key challenge.

Semantic Analysis

Operates at lexical (WSD), sentential (SRL), and discourse (co-reference) levels. Syntax ≠ semantics — "colorless green ideas" is proof.

Tokenization

Deterministic strategies range from whitespace splitting to regex and rule-based approaches. Edge cases (contractions, abbreviations) require explicit handling.

Case-Folding

Reduces vocabulary by collapsing case variants. Harmful for NER, acronym disambiguation, and any task requiring proper noun detection.

Porter Stemmer

5-step heuristic suffix stripper. Fast, simple, but produces non-words. Overstemming and understemming are its primary failure modes.

Lemmatization

Lexicon-based, POS-aware. Always returns valid dictionary words. Handles irregulars ("mice→mouse"). Slower but more accurate than stemming.

Stop-Word Filtration

Removes high-frequency, low-information function words. Great for IR and topic modelling. Dangerous for sentiment (negation) and QA (question words).

The Core Principle

Every preprocessing choice is task-dependent. No universal strategy exists. Always ask: "Does this preprocessing preserve the information my task requires?"

Revision

Quick Revision Notes

  • The NLP pipeline transforms raw text through morphological, syntactic, and semantic layers sequentially.
  • A morpheme is the smallest meaningful unit of language; inflection changes form, derivation changes meaning/category.
  • Constituency parsing produces hierarchical phrase trees (S→NP VP); dependency parsing produces typed word-pair graphs.
  • "Colorless green ideas sleep furiously" — syntactically valid but semantically nonsensical. Levels are distinct.
  • Tokenization must handle contractions, abbreviations, hyphens, URLs, and non-whitespace-delimited languages.
  • Python's casefold() is preferred over lower() for Unicode-correct case folding (e.g., German ß → ss).
  • Case-folding collapses "Turkey" (country) and "turkey" (bird) — destroys named entity boundaries.
  • Porter Stemmer uses word measure m to condition rules. Steps 1a→1b→1c→2→3→4→5a→5b in order.
  • Stemming produces stems (possibly non-words); lemmatization produces lemmas (always valid dictionary words).
  • Lemmatization requires POS tag for correct results: "better" + ADJ → "good"; "better" + VERB → "better".
  • WordNet is organized into synsets (synonym sets) with semantic relations: hypernymy, hyponymy, meronymy, antonymy.
  • Stop-word removal: 30–50% token reduction, but "not bad" → "bad" after removing "not" — sentiment reversal!
  • NLTK has 179 English stop words; spaCy has ~326; always consider domain-specific additions.
  • TF-IDF naturally downweights high-frequency terms — stop-word removal may be redundant with TF-IDF models.
  • Modern LLMs (BERT, GPT) use subword tokenization (BPE/WordPiece) — neither stemming nor stop-word removal is applied.
  • Error propagation: mistakes at tokenization level cascade and amplify through every downstream pipeline stage.
Common Mistakes

Common Misconceptions & Mistakes

MisconceptionReality
"Stemming and lemmatization are the same thing" Stemming is heuristic suffix stripping; lemmatization is linguistically informed dictionary lookup. They produce different outputs and are suited to different tasks.
"Always remove stop words for better performance" Stop-word removal destroys critical information for sentiment analysis (negation), QA (question words), and discourse tasks. It is task-dependent.
"Case-folding is always safe" Case-folding conflates proper nouns with common nouns (Turkey/turkey, Apple/apple) and destroys acronym identity (IT/it). It is harmful for NER.
"Syntax and semantics are the same level" Syntax is about grammatical structure; semantics is about meaning. A sentence can be syntactically correct but semantically anomalous, and vice versa.
"The Porter Stemmer outputs valid English words" The Porter Stemmer regularly produces non-words: "studies" → "studi", "happiness" → "happi". These are stems, not lemmas.
"Tokenization is trivial — just split on spaces" Whitespace tokenization fails badly on contractions, abbreviations, numbers, URLs, multilingual text, and punctuation. Robust tokenization requires explicit rule handling.
"Lemmatization doesn't need POS tags" Without POS tags, WordNet defaults to treating all words as nouns, severely degrading lemmatization quality for verbs and adjectives.
"Morphological analysis is irrelevant for deep learning" While neural models encode morphology implicitly, understanding morphological analysis is essential for interpreting model behaviour, diagnosing failures, and working in low-resource or morphologically complex languages.
